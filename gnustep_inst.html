Guia para instalar GNUstep en sistemas GNU/Linux

Los siguientes paquetes son requisitos para poder instalar GNUstep (los nombres pueden cambiar con el sistema operativo):

  • libffi-dev
  • libssl-dev
  • libxml2-dev
  • libxslt1-dev
  • libjpeg62-dev
  • libtiff4-dev
  • libpng12-dev
  • libungif4-dev
  • libart-2.0-dev
  • libcairo2-dev
  • libfreetype6-dev
  • libx11-dev
  • libxft-dev
  • libxmu-dev

La forma más fácil de instalar GNUstep es con el paquete gnustep-startup. La última versión puede obtenerse aquí. Una vez que el arhivo ha sido extraido, en una terminal vamos a la carpeta extraida y, con permisos de root:

./InstallGNUstep

Esto inicia la instalación de GNUstep. Terminada la instalación, solo falta asegurar el inicio de los servicios de GNUstep al arrancar el sistema. Para ello en el archivo rc.local, generalmente ubicado en la ruta /etc, agregamos el siguiente código antes de exit 0 (se necesitan permisos de root):

GNUSTEP_SYSTEM_ROOT=/usr/GNUstep/System

if [ -f $GNUSTEP_SYSTEM_ROOT/Tools/gdomap ]; then

$GNUSTEP_SYSTEM_ROOT/Tools/gdomap

fi

Ahora, en el archivo .profile o .bash_profile de tu carpeta personal (observa que estos son archivos ocultos) necesitas agregar el siguiente código:

. /usr/GNUstep/System/Library/Makefiles/GNUstep.sh

gdnc

gpbs

make_services

Con esto realizado, al iniciar tu sesion los servicios gdomap, gdnc y gpbs deben estar corriendo. Tal vez desees instalar también la aplicación System Preferences para configurar GNUstep. Dicha aplicación se puede obtener aquí.
